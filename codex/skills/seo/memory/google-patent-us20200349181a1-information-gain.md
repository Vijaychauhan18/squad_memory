# Core Concept
`US20200349181A1` formalizes information gain: how much net-new value a document adds beyond documents the user has already seen.

## Official Patent Record
- Publication: `US20200349181A1`
- Granted family member: `US11354342B2`
- Title: `Contextual estimation of link information gain`
- Prior art date: `2018-10-18`
- Filing date: `2018-10-18`
- Publication date: `2020-11-05`
- Assignee: `Google LLC`
- Inventors: `Victor Carbune`, `Pedro Gonnet Anders`
- Legal status: `Granted / Active`

## Practical Mechanism
- The system keeps a first set of already-seen documents and a second set of candidate documents on the same topic.
- It estimates an information gain score for each candidate based on what additional information it contributes beyond the first set.
- The score can be produced by machine learning over full document content, extracted salient information, embeddings, or other document representations.
- Candidate documents can be reranked as the user's viewed set changes.

## SEO Reading
- This is a direct anti-copycat signal. Commodity content that repeats what the user already consumed is less valuable than content with new comparisons, edge cases, or evidence.
- Patent Core's reading is especially useful for AI-search strategy because answer systems need chunks that add distinct value, not interchangeable summaries.
- "Helpful" content is not just complete; it is incrementally useful in context.

## What To Reuse
- Publish pages that add original frameworks, examples, data, tests, or contradictions, not just complete coverage.
- Audit overlapping pages by asking what each page contributes that the rest of the cluster does not.
- Structure content so the novel sections are easy to extract into passages and summaries.
- Use this note when planning updates to over-saturated SERPs or AI-answer-targeted content.

## Small Details Worth Remembering
- The patent assumes a first set of already-consumed documents and a second set of candidate documents, then scores candidates against the user's existing information state.
- Inputs can include full text, salient extracted content, embeddings, bags of words, and other document representations.
- Information gain can be recalculated as the user consumes more content, so the ranking space is dynamic, not fixed.
- The family is useful for both search results and assistant-style summarized responses.

## E-E-A-T And Audit Lens
- Expertise is stronger when the page adds new methods, evidence, examples, or interpretation rather than rephrasing commodity content.
- Experience often shows up as incremental detail others do not have: tests, screenshots, field observations, failure cases, comparisons.
- Low-gain content may still be relevant, but it is less likely to be selected when systems need to pick one passage or one answer.

## Real Audit Questions
- What does this page add that a user probably did not already get from the top results?
- Which sections are genuinely net-new versus reworded SERP consensus?
- Does the page offer proprietary evidence, fresh synthesis, or operational guidance?
- If an AI system had to cite one chunk from this page, which chunk is the information-gain winner?

## Caveat
Information gain is contextual. A page can be valuable for one user state and redundant for another, so this is not a blanket novelty score.
