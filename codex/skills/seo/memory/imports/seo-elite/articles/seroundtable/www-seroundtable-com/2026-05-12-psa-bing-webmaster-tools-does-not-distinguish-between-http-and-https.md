# PSA: Bing Webmaster Tools Does Not Distinguish Between http and https

## Why This Matters
Did you know that Bing Webmaster Tools treats http and https as the same property, it does not distinguish between http and https? This is different from Google Search Console, which lets you set up a property for each and every variation.

## Extracted Article Passages
- Did you know that Bing Webmaster Tools treats http and https as the same property, it does not distinguish between http and https? This is different from Google Search Console, which lets you set up a property for each and every variation.
- We covered this in 2020 , but for some reason, that tweet from the Bing team was deleted.
- In any event, Krishna Madhavan from the Bing Webmaster Tools team wrote on LinkedIn , "From a Bing Webmaster Tools perspective, we don’t distinguish between http and https, so they effectively appear as the same property to customers regardless of what’s imported."
- So when you import your Google Search Console properties to Bing Webmaster Tools, keep this in mind.
- I agree that setups where there are different content for http and https are super rare (and generally problematic). From a Bing Webmaster Tools perspective, we don’t distinguish between http and https, so they effectively appear as the same property to customers regardless of what’s imported. We typically expect https (given where the web has moved), but if that data isn’t available, we should fall back to http or other ports.
